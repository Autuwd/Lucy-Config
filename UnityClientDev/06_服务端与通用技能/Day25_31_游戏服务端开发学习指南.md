## 阶段八：服务端核心基础（4 天）

### 第 25 天：Linux 操作系统与 Shell/Python 脚本
*   **核心概念：**
    *   Linux 常用命令（`ls`, `cd`, `ps`, `top`, `netstat`, `lsof`, `tail -f`）
    *   文件权限与进程管理（`chmod`, `chown`, `kill -9`）
    *   Shell 自动化脚本（进程启停、基础备份）
    *   Python 服务端辅助脚本（多日志分析、Pandas 数据统计）
*   **代码示例：服务启停 Shell 脚本 (run.sh)**
    ```bash
    #!/bin/bash
    APP_NAME="GameServer"
    PID=$(ps -ef | grep $APP_NAME | grep -v grep | awk '{print $2}')

    case "$1" in
        start)
            if [ -z "$PID" ]; then
                nohup ./$APP_NAME > /dev/null 2>&1 &
                echo "$APP_NAME started."
            else
                echo "$APP_NAME is already running (PID: $PID)."
            fi
            ;;
        stop)
            if [ -z "$PID" ]; then
                echo "$APP_NAME is not running."
            else
                kill -15 $PID
                echo "$APP_NAME stopped gracefully."
            fi
            ;;
        *)
            echo "Usage: $0 {start|stop}"
            ;;
    esac
    ```

### 第 26 天：高性能网络编程与 Socket 模型
*   **核心概念：**
    *   TCP、UDP 与 HTTP 在服务端的应用场景与连接吞吐优化
    *   I/O 多路复用机制（Select, Poll, Epoll 的核心原理与性能差异）
    *   阻塞 I/O (Multi-threaded) 与非阻塞 I/O (Reactor 反应器模式)
    *   消息协议批量制订与生成（Protobuf 序列化与反序列化，防粘包/半包设计）
*   **C++ 示例：Epoll 核心伪代码**
    ```cpp
    #include <sys/epoll.h>
    #include <unistd.h>

    void StartEpollServer() {
        int server_fd = CreateAndBindSocket();
        listen(server_fd, SOMAXCONN);

        int epoll_fd = epoll_create1(0);
        struct epoll_event ev, events[64];
        ev.events = EPOLLIN;
        ev.data.fd = server_fd;
        epoll_ctl(epoll_fd, EPOLL_CTL_ADD, server_fd, &ev);

        while (true) {
            int nfds = epoll_wait(epoll_fd, events, 64, -1);
            for (int i = 0; i < nfds; ++i) {
                if (events[i].data.fd == server_fd) {
                    int client_fd = accept(server_fd, nullptr, nullptr);
                    SetNonBlocking(client_fd);
                    ev.events = EPOLLIN | EPOLLET; // 边缘触发
                    ev.data.fd = client_fd;
                    epoll_ctl(epoll_fd, EPOLL_CTL_ADD, client_fd, &ev);
                } else {
                    HandleClientData(events[i].data.fd);
                }
            }
        }
    }
    ```

### 第 27 天：数据库与内存缓存设计 (MySQL + Redis)
*   **核心概念：**
    *   关系型数据库 MySQL 的索引优化、事务级别与读写分离
    *   内存数据库 Redis 常用数据类型及配置（String, Hash, List, Set, Sorted Set）
    *   应用层缓存系统（Write-Back 延迟回写，热数据内存驻留，防止频繁读写磁盘）
    *   游戏数据备份与容灾机制

### 第 28 天：多线程、协程与无锁进程框架
*   **核心概念：**
    *   多线程模型（主线程、网络I/O线程、数据库线程、日志线程、线程间无锁队列通信）
    *   异步非阻塞逻辑框架（利用 协程/Coroutine 或 回调系统 消除嵌套异步）
    *   多进程架构设计（Gate网关进程、World逻辑进程、DB数据进程、Chat聊天进程及 IPC 消息总线）

## 阶段九：游戏服务端逻辑与高可用开发（3 天）

### 第 29 天：游戏逻辑与核心系统开发
*   **核心概念：**
    *   **养成系统：** 背包、装备、任务等核心养成逻辑，保障数据 ACID
    *   **战斗系统：** 状态同步与帧同步在服务端的判定校验，防外挂伤害公式校验
    *   **社交与军团：** 跨服军团、全局排行榜逻辑（Redis Sorted Set 极其高效的 O(log N) 排名实现）

### 第 30 天：容错、可靠性与架构扩展
*   **核心概念：**
    *   丢包处理、重连机制与延迟补偿（无状态服务设计）
    *   **分服设计与平滑扩展：** 动态合服、单服上限管理及分布式节点注册
    *   数据校验防作弊（客户端消息合理性检查，防刷金币、物理碰撞双重校验）

### 第 31 天：服务端运维与 GM 管理工具
*   **核心概念：**
    *   **日志分析系统：** 提取 run/db 日志计算留存率、付费率、ARPU、LTV，监控玩家道具/英雄分布
    *   **GM 管理后台：** 基于 Python Django 或 Web 框架的网页操作，实现补偿邮件、全服公告、玩家禁言/拉黑
    *   **服务器监控：** 进程监控、性能报警、平滑热更更新
