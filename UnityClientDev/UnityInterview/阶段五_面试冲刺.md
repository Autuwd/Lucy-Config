## 5.1 项目经验整理

#### 核心概念解析

**项目经验在面试中的重要性**
- 项目经验是面试的核心考察点，占面试总分的 40-60%
- 面试官通过项目了解你的**实际开发能力**和**问题解决能力**
- 好的项目经验可以**弥补学历或经验上的不足**

```
面试官看项目的三个层次：

第一层：做什么（What）
- 项目是什么类型的游戏
- 使用了什么技术
- 属于什么规模

第二层：怎么做（How）
- 核心功能如何实现
- 遇到了什么问题
- 如何解决的

第三层：为什么这样做（Why）
- 为什么选择这个方案
- 有什么更好的替代方案
- 从中学到了什么
```

**项目选择的原则**

| 原则 | 说明 | 示例 |
|------|------|------|
| 完整性 | 有完整的游戏流程 | 开始→进行→结束 |
| 复杂度 | 有一定技术难度 | AI、寻路、网络同步 |
| 可展示性 | 能现场演示 | APK、EXE |
| 真实性 | 真正动手做过 | 不是抄的 |

**如何描述项目亮点**

| 类型 | 描述方式 | 效果 |
|------|----------|------|
| 性能优化 | 优化前数据 → 优化后数据 | 直观体现能力 |
| 难点解决 | 问题→方案→结果 | 体现解决问题能力 |
| 架构设计 | 用了什么模式、为什么 | 体现抽象思维 |
| 团队协作 | 分工→协作→成果 | 体现软技能 |

### 项目选择原则

```
至少准备 1-2 个完整项目

好的项目包含：
✓ 完整的游戏流程（开始→游戏→结束）
✓ 有技术挑战（性能优化/AI/网络同步）
✓ 有代码架构（设计模式/MVC）
✓ 有可运行版本（APK/EXE）

推荐项目类型：
- 塔防游戏（适合展示多种能力）
- 多人联机游戏（展示网络能力）
- 动作游戏（展示物理/动画）
```

## 5.4 模拟面试问题

### C# 基础问题

#### Q1: ref 和 out 的区别？

```
参考回答：
ref 和 out 都是引用传递，但有区别：
1. ref 必须在传入前初始化，out 不需要
2. out 必须在方法内赋值，ref 不需要
3. out 通常用于返回多个值，如 TryParse

代码示例：
void RefDemo(ref int x) { x = 10; }
void OutDemo(out int x) { x = 10; }

int a;
OutDemo(out a);  // 正确
RefDemo(ref a);  // 错误，a 未初始化
```

#### Q2: 什么是 GC？

```
参考回答：
GC 是垃圾回收器，自动管理托管内存。
.NET 的 GC 是分代的：Gen0、Gen1、Gen2。
新对象在 Gen0，回收最频繁；
存活久的对象会升级到 Gen2。
GC 会暂停所有托管线程，所以要减少 GC。
```

#### Q3: async/await 原理？

```
参考回答：
async/await 是语法糖，基于 Task。
编译器会将代码转换为状态机。
await 会捕获当前上下文，异步完成后继续执行。
不会阻塞线程，适合 I/O 操作。
CPU 密集型用 Task.Run 包装。
```

### Unity 进阶问题

#### Q4: Unity 生命周期？

```
参考回答：
Awake → OnEnable → Start
↓ 
FixedUpdate（物理）
↓ 
Update（逻辑）
↓ 
LateUpdate（相机跟随）
↓
OnDisable → OnDestroy
```

#### Q5: GetComponent 性能？

```
参考回答：
GetComponent 有开销，每帧调用会累积。
应该在 Awake/Start 中缓存引用。
Unity 6 可以用 readonly 字段自动缓存。
```

#### Q6: 性能优化手段？

```
渲染优化：
- 静态批处理
- 动态批处理
- GPU Instancing
- 图集
- 遮挡剔除

内存优化：
- 对象池
- 减少字符串拼接
- 及时销毁资源

代码优化：
- 缓存组件引用
- 减少 LINQ
- for 替代 foreach
```

### 新增八股模拟问题

#### 操作系统/计算机组成

**Q: 谈谈你对 CPU 缓存的理解，以及它在游戏开发中的影响？**
```
答题框架：
1. 缓存层级：L1(~1ns) → L2(~5ns) → L3(~15ns) → RAM(~100ns)
2. 缓存行概念：一次性加载 64 字节连续数据
3. 影响：
   - 数组/List<T> 遍历快于 LinkedList（连续 vs 离散）
   - Unity ECS 按组件连续存储，提高缓存命中率
   - Job System 中数据布局直接影响性能
4. C++ 对比：alignas(64) 避免伪共享，std::vector 替代 std::list
```

**Q: 进程和线程有什么区别？Unity 中怎么使用线程？**
```
答题框架：
1. 进程=资源分配单位，线程=CPU调度单位
2. 进程隔离，线程共享地址空间
3. Unity 线程模型：主线程 + 渲染线程 + Job Worker
4. 限制：子线程不能调用 Unity API（Transform、GameObject 等）
5. 正确做法：耗时计算用 Job System，结果通过 MainThread 回调更新
6. C++ 对比：CreateProcess/CreateThread vs std::thread
```

#### 计算机网络

**Q: 设计一个多人游戏的网络同步方案，TCP 还是 UDP？**
```
答题框架：
1. 区分业务场景：
   - 登录/匹配/聊天 → TCP（可靠）
   - 实时位置同步 → UDP（低延迟）
2. 帧同步 vs 状态同步：
   - 帧同步（UDP，如《王者荣耀》）：只同步输入，所有客户端同步运算
   - 状态同步（TCP，如 MMO）：服务器计算，客户端只接收结果
3. Unity 选型：Photon（UDP 可靠）、Mirror（TCP）、LiteNetLib（UDP+自定义可靠层）
```

#### C++ 对比

**Q: C# 和 C++ 在内存管理上有什么本质区别？**
```
答题框架：
1. C# 有 GC（自动回收）、C++ 手动（delete/智能指针）
2. C# 所有引用类型在堆上分配，C++ 可栈可堆
3. GC 好处：避免内存泄漏；GC 坏处：STW 暂停，不适合实时系统
4. Unity 优化：对象池、减少 string 拼接、struct 替代 class
5. C++ 方案：RAII（析构自动释放）、智能指针（shared_ptr/unique_ptr）
```
