## 六、紧急背诵清单

### 这些必须背下来

1. **生命周期顺序**：Awake → OnEnable → Start → FixedUpdate → Update → LateUpdate → OnDisable → OnDestroy

2. **ref vs out**：
   - ref：传入前必须赋值，双向传递
   - out：不需要赋值，方法内必须赋值，单向传出

3. **碰撞 vs 触发**：
   - 碰撞：需要两个 Collider + 至少一个 Rigidbody
   - 触发：需要至少一个 Rigidbody，isTrigger = true

4. **对象池核心优势**：避免频繁创建销毁，减少 GC，提升性能

5. **DrawCall 优化**：静态批处理、动态批处理、GPU Instancing、图集

## 九、面试核心问题及简要解答（简明要点，便于笔记粘贴）

Q1: ref 和 out 的区别？

- 要点
  - ref：调用前必须初始化；方法内对参数的修改会反映到调用处的变量上
  - out：调用前不需要初始化；方法内必须赋值，返回一个值给调用者
- 参考要点示例
  - ref 示例
    - void Increment(ref int x) { x += 1; }
    - int a = 5; Increment(ref a); // a=6
  - out 示例
    - void GetMinMax(int[] arr, out int min, out int max) { min = arr[0]; max = arr[0]; foreach (var n in arr) { if (n < min) min = n; if (n > max) max = n; } }
    - int mn, mx; GetMinMax(new int[]{3,1,4,2}, out mn, out mx); // mn=1, mx=4

Q2: 什么是多态？如何实现？

- 要点
  - 多态：同一接口，不同实现；运行时根据对象类型执行对应的方法
  - 实现方式：通过虚方法（virtual/override）、抽象方法（abstract）、接口（interface）实现
- 简例
  - class Animal { public virtual void Speak() { Console.WriteLine("Animal sound"); } }
  - class Dog : Animal { public override void Speak() { Console.WriteLine("Bark"); } }
  - Animal a = new Dog(); a.Speak(); // 输出 "Bark"

Q3: 接口和抽象类的区别？

- 要点
  - 接口是行为契约，没有实现；抽象类可以有实现、字段、构造等
  - 类只能继承一个抽象类，可以实现多个接口
  - 场景：接口用于行为契约，抽象类用于代码复用和默认实现
- 要点示例
  - interface IDamageable { void TakeDamage(int d); }
  - abstract class Character { public virtual void Move() {} public abstract void Attack(); }

Q4: Unity 生命周期顺序？

- 要点
  - 常见顺序：Awake → OnEnable → Start → Update → LateUpdate → OnDisable → OnDestroy
  - FixedUpdate 适合物理相关
- 备注：Awake 在对象实例化时调用；OnEnable 当对象启用时调用；Start 在第一帧更新前调用

Q5: Update、FixedUpdate、LateUpdate 区别？

- 要点
  - Update：每帧更新
  - FixedUpdate：与物理步长绑定，适合物理相关的更新
  - LateUpdate：在所有 Update 完成后执行，适合摄像机跟随等需要“最终位置”的更新

Q6: 碰撞和触发的区别？

- 要点
  - 碰撞：两者都参与，至少一个 Rigidbody，事件是 CollisionEnter/Stay/Exit
  - 触发（Trigger）：任一 Collider 设为 isTrigger，事件是 TriggerEnter/Stay/Exit
- 设计要点：若需要物理交互，选碰撞；若仅触发事件，选触发

Q7: GC 与优化？

- 要点
  - GC 管理托管内存，分代 GC，频繁分配导致帧率波动
  - 优化：减少分配、对象池、缓存引用、避免 Update 中分配、使用 StringBuilder 替代字符串拼接

Q8: 对象池原理？

- 要点
  - 通过复用对象，减少分配/销毁，降低 GC 峰值
  - 关键点：Get/Return、预加载、容量控制、回收/重置逻辑

Q9: 协程 vs 线程？

- 要点
  - 协程：在主线程中分步执行，不是真正的多线程
  - 线程：后台执行，需避免直接调用 Unity API，UI/场景操作应在主线程完成
- 实践要点：耗时操作放后台，UI/场景更新仍在主线程

Q10: 快速排序原理？

- 要点
  - 分治策略：选取基准，将数组分为小于与大于两部分，递归排序
- 简要要点：分区、递归、边界条件

Q11: 二分查找原理？

- 要点
  - 在有序数组中逐步缩小区间，比较中点并移动左右边界，直到找到目标或区间为空
- 要点提示：边界初始化、循环条件、避免越界

Q12: 单例模式实现？

- 要点
  - 私有化构造、静态实例、公开的 Instance 获取入口
  - 在 Unity 中通常使用 DontDestroyOnLoad 维护全局实例，需处理重复实例的情况

### 新增八股专题（计算机基础）

**计算机组成原理 🔴🔴**
Q13: CPU 三级缓存与内存关系？
- 要点
  - 寄存器→L1→L2→L3→RAM→SSD，速度递减、容量递增
  - 缓存行（Cache Line，通常64字节）是缓存与内存交换的最小单位
  - 空间局部性：数组遍历快于链表
  - Unity：Job System 利用连续内存（NativeArray）提高缓存命中率
- C++ 对比：`alignas(64)` 控制对齐、`__builtin_prefetch` 预取

Q14: 什么是伪共享（False Sharing）？
- 要点
  - 多核修改同一缓存行的不同变量，导致缓存失效和性能下降
  - 解决方案：缓存行填充（padding）
- Unity 场景：Job System 中不同 Job 的数据尽量独立

**操作系统 🔴🔴🔴**
Q15: 进程和线程的区别？
- 要点
  - 进程是资源分配最小单位，线程是 CPU 调度最小单位
  - 进程独立地址空间，线程共享进程地址空间
  - 切换开销：进程重、线程轻
  - Unity：主线程 + 渲染线程 + Job Worker 线程；子线程不能调用 Unity API

Q16: 死锁的四个必要条件？
- 要点
  - 互斥、持有并等待、不可剥夺、循环等待
  - 预防：破坏任一条件，如按序加锁、超时机制
  - Unity：lock 语句、Monitor.TryEnter 避免死锁

Q17: 用户态 vs 内核态？
- 要点
  - 内核态可访问所有硬件资源，用户态受限
  - 系统调用（如文件读写、网络请求）触发用户态→内核态切换
  - Unity 性能：减少系统调用，使用对象池替代频繁 new/delete

**计算机网络 🔴🔴**
Q18: TCP 三次握手与四次挥手？
- 要点
  - 三次握手：SYN → SYN-ACK → ACK（建立连接）
  - 四次挥手：FIN → ACK → FIN → ACK（断开连接）
  - Unity 多人游戏：Photon 使用 UDP（可靠通道模拟 TCP），Mirror 用 TCP

Q19: TCP 和 UDP 的区别及 Unity 中的应用？
- 要点
  - TCP 可靠保序但延迟高，适合状态同步（MMO 角色移动）
  - UDP 不可靠但速度快，适合帧同步（MOBA、FPS 即时操作）
  - Unity 方案：Mirror（TCP）、LiteNetLib（UDP+可靠层）、Photon（TCP/UDP 混合）

Q20: HTTP 状态码分类？
- 要点
  - 1xx 信息、2xx 成功（200）、3xx 重定向（301/302）、4xx 客户端错误（404/403）、5xx 服务器错误（500/502）
  - Unity 网络请求：UnityWebRequest 处理各状态码反馈
