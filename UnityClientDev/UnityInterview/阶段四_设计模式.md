## 4.1 单例模式

#### 核心概念解析

**什么是单例模式？**
- 单例模式确保一个类**只有一个实例**，并提供**全局访问点**
- 是最简单、最常用的设计模式之一

```
单例模式示意图：

┌─────────────────────────────────────────────────────┐
│  普通类（每次 new 是新对象）                          │
│  ┌─────────┐                                         │
│  │ new A() │ → obj1                                  │
│  ├─────────┤                                         │
│  │ new A() │ → obj2  ← 不同对象！                   │
│  └─────────┘                                         │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│  单例类（始终是同一个对象）                           │
│  ┌─────────┐                                         │
│  │ A.Instance│ → obj1  ← 始终返回同一对象           │
│  ├─────────┤                                         │
│  │ A.Instance│ → obj1  ← 还是同一个！              │
│  └─────────┘                                         │
└─────────────────────────────────────────────────────┘
```

**单例模式的要点**
1. **私有构造函数**：禁止外部 `new` 创建对象
2. **静态实例**：类级别保存唯一实例
3. **公开访问点**：通过 `Instance` 属性获取实例
4. **线程安全**（多线程环境）：防止并发创建多个实例

**单例模式的优缺点**

| 优点 | 缺点 |
|------|------|
| 全局唯一，避免传递参数 | 耦合度高，难以测试 |
| 访问方便 | 违反单一职责原则 |
| 懒加载/饿汉式可选 | 可能导致内存泄漏 |

**Unity 中单例的使用场景**
- GameManager：游戏全局管理
- AudioManager：音频管理
- UIManager：UI 管理
- ResourceManager：资源管理

### 基础单例

```csharp
public class GameManager : MonoBehaviour
{
    // 单例实例
    public static GameManager Instance { get; private set; }
    
    void Awake()
    {
        // 检查是否已存在实例
        if (Instance != null && Instance != this)
        {
            Destroy(gameObject);
            return;
        }
        
        Instance = this;
        
        // 跨场景保持
        DontDestroyOnLoad(gameObject);
    }
}
```

### 泛型单例基类

```csharp
// 单例基类（注意：Unity API 只能在主线程调用，无需 lock）
public class Singleton<T> : MonoBehaviour where T : MonoBehaviour
{
    private static T instance;
    
    public static T Instance
    {
        get
        {
            if (instance == null)
            {
                instance = FindObjectOfType<T>();
                
                if (instance == null)
                {
                    GameObject singletonObj = new GameObject(typeof(T).Name);
                    instance = singletonObj.AddComponent<T>();
                }
            }
            return instance;
        }
    }
    
    protected virtual void Awake()
    {
        if (instance == null)
        {
            instance = this as T;
            DontDestroyOnLoad(gameObject);
        }
        else if (instance != this)
        {
            Destroy(gameObject);
        }
    }
}

// 使用
public class AudioManager : Singleton<AudioManager>
{
    protected override void Awake()
    {
        base.Awake();
        // AudioManager 特有初始化
    }
}
```

## 4.2 观察者模式

#### 核心概念解析

**什么是观察者模式？**
- 观察者模式定义了**一对多**的依赖关系
- 当一个对象（主题/被观察者）状态变化时，所有依赖它的对象（观察者）都会收到通知

```
观察者模式示意图：

┌───────────────────┐          ┌───────────────────┐
│   主题 (Subject)   │         │   观察者 A        │
│  ─────────────    │         │  ─────────────    │
│  + 注册观察者()   │────────→│  + OnNotify()    │
│  + 移除观察者()   │         │                   │
│  + 通知观察者()   │         └───────────────────┘
│                   │         ┌───────────────────┐
└───────────────────┘────────→│   观察者 B        │
                               │  ─────────────    │
                               │  + OnNotify()    │
                               └───────────────────┘

例子：玩家死亡时，所有订阅了"玩家死亡事件"的系统都会收到通知
- UI 系统：显示"游戏结束"
- 音效系统：播放死亡音效
- 统计系统：记录死亡次数
```

**观察者模式的组成**

| 角色 | 描述 | Unity 中对应 |
|------|------|-------------|
| Subject（主题） | 被观察的对象，维护观察者列表 | 事件中心、EventSystem |
| Observer（观察者） | 接收通知的对象 | 订阅事件的脚本 |
| Notify（通知） | 状态变化时触发 | 事件触发 |

**观察者模式 vs 事件/委托**

| 特性 | 观察者模式 | 事件/委托 |
|------|-----------|----------|
| 耦合度 | 松耦合 | 更松散 |
| 实现复杂度 | 较高 | 简单（C# 事件） |
| 适用场景 | 复杂的一对多 | 简单通知 |

**Unity 中的观察者实现**
- C# 事件（最常用）
- UnityEvent（可可视化配置）
- 第三方事件系统

### 什么是观察者模式？

定义对象间的一对多依赖，当一个对象状态改变时，所有依赖它的对象都会收到通知

### 事件系统实现

```csharp
// 事件参数
public class HealthChangedEventArgs : System.EventArgs
{
    public int oldHealth;
    public int newHealth;
}

// 事件发布者
public class Player : MonoBehaviour
{
    // 定义事件
    public event System.EventHandler<HealthChangedEventArgs> OnHealthChanged;
    public event System.Action OnDied;
    
    private int health = 100;
    public int Health
    {
        get => health;
        private set
        {
            if (health != value)
            {
                int oldHealth = health;
                health = value;
                
                // 触发事件
                OnHealthChanged?.Invoke(this, new HealthChangedEventArgs 
                { 
                    oldHealth = oldHealth, 
                    newHealth = health 
                });
                
                if (health <= 0)
                {
                    OnDied?.Invoke();
                }
            }
        }
    }
    
    public void TakeDamage(int damage)
    {
        Health -= damage;
    }
}

// 观察者1：UI
public class PlayerUI : MonoBehaviour
{
    public Player player;
    
    void OnEnable()
    {
        player.OnHealthChanged += OnHealthChangedHandler;
        player.OnDied += OnDiedHandler;
    }
    
    void OnDisable()
    {
        player.OnHealthChanged -= OnHealthChangedHandler;
        player.OnDied -= OnDiedHandler;
    }
    
    void OnHealthChangedHandler(object sender, HealthChangedEventArgs e)
    {
        Debug.Log($"UI: 血量变化 {e.oldHealth} -> {e.newHealth}");
    }
    
    void OnDiedHandler()
    {
        Debug.Log("UI: 玩家死亡");
    }
}

// 观察者2：音效
public class PlayerAudio : MonoBehaviour
{
    public Player player;
    
    void OnEnable()
    {
        player.OnHealthChanged += PlayDamageSound;
    }
    
    void OnDisable()
    {
        player.OnHealthChanged -= PlayDamageSound;
    }
    
    void PlayDamageSound(object sender, HealthChangedEventArgs e)
    {
        if (e.newHealth < e.oldHealth)
        {
            Debug.Log("播放受伤音效");
        }
    }
}
```

## 4.3 状态机模式

#### 核心概念解析

**什么是状态机？**
- 状态机（State Machine）管理对象的不同**状态**及状态之间的**转换**
- 每个状态有特定的行为，状态之间有明确的转换条件

```
状态机示意图（玩家角色）：

        ┌──────────┐
        │   Idle   │  ← 待机状态
        │  站立/呼吸│
        └────┬─────┘
             │ 按下移动键
             ▼
        ┌──────────┐
        │  Running │  ← 奔跑状态
        │  播放动画 │
        └────┬─────┘
             │ 松开移动键
             ▼        │ 按下跳跃键
        ┌──────────┐  │
        │  Jumping │←─┘
        │  跳跃动画 │
        └────┬─────┘
             │ 落地
             ▼
        ┌──────────┐
        │   Idle   │  → 返回待机
        └──────────┘
```

**有限状态机（FSM）的要素**

| 要素 | 说明 | 示例 |
|------|------|------|
| State（状态） | 对象当前的状态 | Idle、Run、Jump |
| Event/Transition | 触发状态转换的事件 | 按下跳跃键 |
| Condition | 转换条件 | 是否在地面 |
| Action | 转换时执行的动作 | 播放动画、加力 |

**状态机的实现方式**

| 方式 | 优点 | 缺点 |
|------|------|------|
| switch-case | 简单直观 | 难以扩展 |
| 状态模式 | 符合开闭原则 | 代码量大 |
| 状态机插件 | 功能强大 | 学习成本 |

**状态机的应用场景**
- 玩家角色状态管理
- AI 敌人行为（巡逻、追击、攻击）
- 游戏流程（菜单、游戏中、暂停）
- UI 界面状态

### 什么是状态机？

管理对象的不同状态及状态间的转换

### 简单状态机

```csharp
// ========== 状态接口 ==========
// 定义状态的统一接口
// 所有具体状态必须实现这三个方法
public interface IState
{
    // Enter：进入状态时调用
    // 用于：初始化状态数据、播放动画等
    void Enter();
    
    // Update：每帧调用
    // 用于：检测状态转换条件、执行状态行为
    void Update();
    
    // Exit：退出状态时调用
    // 用于：清理数据、保存状态等
    void Exit();
}

// ========== 状态机核心 ==========
// 管理状态的切换和更新
public class StateMachine
{
    // 当前状态
    private IState currentState;
    
    // 只读属性：获取当前状态
    public IState CurrentState => currentState;
    
    // 切换状态
    // 1. 先调用当前状态的Exit（清理工作）
    // 2. 切换到新状态
    // 3. 调用新状态的Enter（初始化工作）
    public void ChangeState(IState newState)
    {
        // ?.：空条件调用，防止currentState为null时报错
        currentState?.Exit();      // 退出旧状态
        currentState = newState;    // 切换到新状态
        currentState?.Enter();      // 进入新状态
    }
    
    // 每帧更新
    // 将更新委托给当前状态处理
    public void Update()
    {
        currentState?.Update();
    }
}

// ========== 待机状态 ==========
public class IdleState : IState
{
    // 持有对PlayerController的引用，用于访问玩家数据和切换状态
    private PlayerController player;
    
    // 构造函数：接收PlayerController实例
    public IdleState(PlayerController player)
    {
        this.player = player;
    }
    
    // 进入状态时播放待机动画
    public void Enter()
    {
        player.PlayAnimation("Idle");
    }
    
    // 每帧检测是否满足状态转换条件
    public void Update()
    {
        // 条件1：玩家开始移动 → 切换到跑步状态
        if (player.IsMoving)
        {
            // 创建新状态并切换
            player.ChangeState(new RunState(player));
        }
        
        // 条件2：按下跳跃键且在地面 → 切换到跳跃状态
        if (Input.GetButtonDown("Jump") && player.IsGrounded)
        {
            player.ChangeState(new JumpState(player));
        }
        
        // 条件3：按下鼠标左键 → 切换到攻击状态
        if (Input.GetMouseButtonDown(0))
        {
            player.ChangeState(new AttackState(player));
        }
    }
    
    // 退出状态时清理（如有需要）
    public void Exit()
    {
    }
}

// ========== 跑步状态 ==========
public class RunState : IState
{
    private PlayerController player;
    
    public RunState(PlayerController player)
    {
        this.player = player;
    }
    
    // 进入状态播放跑步动画
    public void Enter()
    {
        player.PlayAnimation("Run");
    }
    
    // 每帧执行移动逻辑
    public void Update()
    {
        // 执行移动
        player.Move();
        
        // 条件1：玩家停止移动 → 切换回待机
        if (!player.IsMoving)
        {
            player.ChangeState(new IdleState(player));
        }
        
        // 条件2：按下跳跃键 → 切换到跳跃
        if (Input.GetButtonDown("Jump") && player.IsGrounded)
        {
            player.ChangeState(new JumpState(player));
        }
    }
    
    public void Exit()
    {
    }
}

// ========== 玩家控制器 ==========
public class PlayerController : MonoBehaviour
{
    // 状态机实例
    private StateMachine stateMachine;
    
    // 属性：供状态类访问
    public bool IsMoving { get; set; }
    public bool IsGrounded { get; set; }
    
    // Awake：在对象创建时调用（最先执行）
    void Awake()
    {
        // 初始化状态机
        stateMachine = new StateMachine();
    }
    
    // Start：在第一帧Update之前调用
    void Start()
    {
        // 设置初始状态为待机
        ChangeState(new IdleState(this));
    }
    
    // Update：每帧调用
    void Update()
    {
        // 委托给状态机更新
        stateMachine.Update();
    }
    
    // 供状态类调用的方法
    public void ChangeState(IState newState)
    {
        stateMachine.ChangeState(newState);
    }
    
    public void PlayAnimation(string name)
    {
        Debug.Log($"播放动画: {name}");
    }
    
    public void Move()
    {
        Debug.Log("移动");
    }
}
```

```csharp
                if (DetectPlayer()) currentState = State.Chase;
                break;
            case State.Chase:
                Chase();
                if (InAttackRange()) currentState = State.Attack;
                break;
            case State.Attack:
                Attack();
                if (!InAttackRange()) currentState = State.Chase;
                break;
        }
    }
    
    bool DetectPlayer() => Vector3.Distance(transform.position, player.position) < 10f;
    bool InAttackRange() => Vector3.Distance(transform.position, player.position) < 2f;
    void Patrol() {}
    void Chase() {}
    void Attack() {}
}
```

## 4.4 对象池模式

#### 核心概念解析

**什么是对象池模式？**
- 对象池模式是一种管理对象复用的设计模式
- 预先创建一定数量的对象存放在池中，需要时从池获取，用完后归还
- 与第三阶段学习的对象池技术原理相同，是游戏开发中最常用的性能优化模式

```
对象池模式示意图：

┌─────────────────────────────────────────────────────────┐
│                  对象池管理器                            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   ┌─────────────────────────────────────┐               │
│   │           对象池                      │               │
│   │  [Obj1] [Obj2] [Obj3] ... [ObjN]    │               │
│   └─────────────────────────────────────┘               │
│        ↑              ↑              ↑                  │
│    Get()           Return()       预加载                │
│                                                         │
│   优点：                                                │
│   ✓ 减少 new/Destroy 开销                              │
│   ✓ 避免频繁 GC 造成的卡顿                             │
│   ✓ 控制最大对象数量                                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**对象池 vs 直接创建**

| 对比 | 直接创建 | 对象池 |
|------|----------|--------|
| 内存分配 | 每次创建 | 仅初始化时 |
| GC压力 | 高 | 无 |
| 响应速度 | 可能波动 | 稳定 |
| 代码复杂度 | 简单 | 稍复杂 |

**游戏开发中的应用**
- 子弹发射系统
- 粒子特效管理
- 怪物生成系统
- UI元素回收

### 泛型对象池

```csharp
public class ObjectPool<T> where T : class
{
    private readonly Stack<T> pool;
    private readonly System.Func<T> createFunc;
    private readonly System.Action<T> resetAction;
    private readonly int maxSize;
    
    public ObjectPool(System.Func<T> createFunc, 
                      System.Action<T> resetAction = null, 
                      int maxSize = 100)
    {
        this.pool = new Stack<T>();
        this.createFunc = createFunc;
        this.resetAction = resetAction;
        this.maxSize = maxSize;
    }
    
    public T Get()
    {
        if (pool.Count > 0)
        {
            return pool.Pop();
        }
        return createFunc();
    }
    
    public void Return(T obj)
    {
        if (obj == null) return;
        
        resetAction?.Invoke(obj);
        
        if (pool.Count < maxSize)
        {
            pool.Push(obj);
        }
    }
    
    public int Count => pool.Count;
    
    public void Clear()
    {
        pool.Clear();
    }
}

// 使用示例
public class BulletPool : MonoBehaviour
{
    private ObjectPool<Bullet> pool;
    public Bullet bulletPrefab;
    
    void Start()
    {
        pool = new ObjectPool<Bullet>(
            createFunc: () => Instantiate(bulletPrefab),
            resetAction: b => b.Reset(),
            maxSize: 50
        );
    }
    
    public Bullet Get()
    {
        Bullet bullet = pool.Get();
        bullet.gameObject.SetActive(true);
        return bullet;
    }
    
    public void Return(Bullet bullet)
    {
        bullet.gameObject.SetActive(false);
        pool.Return(bullet);
    }
}
```

## 4.5 工厂模式

#### 核心概念解析

**什么是工厂模式？**
- 工厂模式是一种创建型设计模式，将对象的创建逻辑集中管理
- 客户端不需要知道具体产品类的创建过程，只需请求工厂获取
- 符合"开闭原则"，新增产品时无需修改客户端代码

```
工厂模式示意图：

┌─────────────────────────────────────────────────────────┐
│                    工厂模式                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│    ┌──────────────┐      ┌──────────────────┐         │
│    │   工厂类     │─────→│   产品接口        │         │
│    │  + Create() │      └──────────────────┘         │
│    └──────────────┘             ↑                     │
│           │                     │                     │
│           │                     │                     │
│    ┌──────┴──────┐    ┌────────┴────────┐           │
│    │ 产品A       │    │ 产品B             │           │
│    │ (Sword)     │    │ (Bow)             │           │
│    └─────────────┘    └──────────────────┘           │
│                                                         │
│   客户端：factory.Create("Sword") → 获取具体产品        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**工厂模式的类型**

| 类型 | 特点 | 适用场景 |
|------|------|----------|
| 简单工厂 | 一个工厂类创建多种产品 | 产品种类较少 |
| 工厂方法 | 每种产品对应工厂接口 | 产品种类多，需扩展 |
| 抽象工厂 | 创建一系列相关产品 | 产品族概念 |

**为什么游戏开发需要工厂模式？**
- 游戏中经常需要动态创建各种对象（武器、敌人、道具）
- 集中管理创建逻辑，便于更换资源或配置
- 配合对象池使用，实现对象的创建和复用

## 4.6 MVC 架构

#### 核心概念解析

**什么是MVC架构？**
- MVC（Model-View-Controller）是一种经典的分层架构模式
- 将应用程序分为三个核心部分，实现关注点分离
- 便于代码维护和团队协作

```
MVC 架构示意图：

┌─────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────┤

│   │ Model   │←───→│Controller│←───→│  View   │        │
│   │ 数据模型 │     │ 控制器   │     │   视图   │        │
```

**MVC 各部分职责**

| Controller | 协调Model和View，处理输入 | 输入控制、业务逻辑 |

**为什么游戏开发需要MVC？**
- 分离UI和逻辑，便于维护
- 便于团队分工（程序/UI/策划）
- 提高代码复用性

**MVC vs MVP vs MVVM**

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| MVC | Controller处理输入 | Unity传统项目 |
| MVP | View和Model完全解耦 | 复杂UI系统 |
| MVVM | 数据绑定，响应式 | 现代UI框架 |

- 命令模式将请求封装为独立的对象
- 请求包含执行所需的所有信息（接收者、执行动作、参数）

### 什么是命令模式？

将请求封装为对象，从而参数化客户端

```csharp
// 命令接口
public interface ICommand
{
    void Execute();
    void Undo();
}

// 具体命令
public class MoveCommand : ICommand
{
    private readonly CharacterController controller;
    private readonly Vector3 direction;
    private Vector3 lastPosition;
    
    public MoveCommand(CharacterController controller, Vector3 direction)
    {
        this.controller = controller;
        this.direction = direction;
    }
    
    public void Execute()
    {
        lastPosition = controller.transform.position;
        controller.Move(direction * Time.deltaTime);
    }
    
    public void Undo()
    {
        controller.transform.position = lastPosition;
    }
}

public class JumpCommand : ICommand
{
    private readonly Rigidbody rb;
    private float jumpForce;
    private Vector3 velocity;
    
    public JumpCommand(Rigidbody rb, float force)
    {
        this.rb = rb;
        this.jumpForce = force;
    }
    
    public void Execute()
    {
        velocity = rb.velocity;
        rb.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);
    }
    
    public void Undo()
    {
        rb.velocity = velocity;
    }
}

// 命令管理器
public class CommandManager : MonoBehaviour
{
    private Stack<ICommand> commandStack = new Stack<ICommand>();
    
    public void ExecuteCommand(ICommand command)
    {
        command.Execute();
        commandStack.Push(command);
    }
    
    public void Undo()
    {
        if (commandStack.Count > 0)
        {
            ICommand command = commandStack.Pop();
            command.Undo();
        }
    }
    
    public void Clear()
    {
        commandStack.Clear();
    }
}

// 使用
public class PlayerInput : MonoBehaviour
{
    private CommandManager cmdManager;
    private CharacterController cc;
    private Rigidbody rb;
    
    void Start()
    {
        cmdManager = GetComponent<CommandManager>();
        cc = GetComponent<CharacterController>();
        rb = GetComponent<Rigidbody>();
    }
    
    void Update()
    {
        float h = Input.GetAxis("Horizontal");
        float v = Input.GetAxis("Vertical");
        
        Vector3 dir = new Vector3(h, 0, v);
        
        if (dir != Vector3.zero)
        {
            cmdManager.ExecuteCommand(new MoveCommand(cc, dir * 5f));
        }
        
        if (Input.GetButtonDown("Jump"))
        {
            cmdManager.ExecuteCommand(new JumpCommand(rb, 5f));
        }
        
        if (Input.GetKeyDown(KeyCode.Z))
```

## 4.8 享元模式

#### 核心概念解析

**什么是享元模式？**
- 享元模式（Flyweight Pattern）通过共享细粒度对象来减少内存占用
- 核心思想：将对象分为"内部状态"（共享）和"外部状态"（独立）
- 内部状态可以共享，外部状态由客户端管理

```
享元模式示意图：

┌─────────────────────────────────────────────────────────┐
│                    享元模式                              │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   未使用享元：                                          │
│   ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐  ← 每个对象独立         │
│   │ A │ │ A │ │ A │ │ A │ │ A │    占用大量内存         │
│   └───┘ └───┘ └───┘ └───┘ └───┘                         │
│                                                         │
│   使用享元：                                            │
│   ┌───┐ ←── 共享的 A (内部状态)                         │
│   │ A │     ╲                                            │
│   └───┘      ├── 每个位置是独立坐标（外部状态）         │
│   pos1 ─────┘                                          │
│   pos2 ─────┤                                           │
│   pos3 ─────┤                                           │
│   pos4 ─────┤                                           │
│   pos5 ─────┘                                           │
│                                                         │
│   原理：对象属性分为                                    │
│   - 内部状态：可共享的部分（如纹理、模型）              │
│   - 外部状态：各实例不同的部分（如位置、颜色）          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**享元模式的核心角色**

| 角色 | 职责 | 示例 |
|------|------|------|
| Flyweight | 享元接口，定义可共享的方法 | Particle |
| ConcreteFlyweight | 具体享元，实现共享逻辑 | 粒子对象 |
| FlyweightFactory | 享元工厂，管理享元对象池 | 粒子系统 |
| Context | 外部状态，上下文 | 粒子位置、速度 |

**游戏开发中的应用**
- 粒子系统：大量粒子共享纹理和模型
- 树木/草地：场景中大量相似的植物
- 文本渲染：相同字符共享字形数据
- 网络同步：同步对象的共享属性

**享元 vs 对象池**
- 对象池：复用整个对象，节省创建销毁开销
- 享元：对象内部数据共享，减少内存

```csharp
// 1. 撤销/重做
public class CommandManager : MonoBehaviour
{
    private Stack<ICommand> history = new Stack<ICommand>();
    private Stack<ICommand> redoStack = new Stack<ICommand>();
    
    public void Execute(ICommand cmd)
    {
        cmd.Execute();
        history.Push(cmd);
        redoStack.Clear();
    }
    
    public void Undo()
    {
        if (history.Count > 0)
        {
            ICommand cmd = history.Pop();
            cmd.Undo();
            redoStack.Push(cmd);
        }
    }
    
    public void Redo()
    {
        if (redoStack.Count > 0)
        {
            ICommand cmd = redoStack.Pop();
            cmd.Execute();
            history.Push(cmd);
        }
    }
}

// 2. 输入映射命令

    }
}
```

## 4.10 消息总线（Event Bus）

#### 核心概念解析

**什么是消息总线？**
- 消息总线是全局事件系统，替代直接的事件订阅
- 组件间无需直接引用，通过消息中心通信

### 消息总线实现

```csharp
// 消息类型
public class MessageCenter
{
    private static Dictionary<string, System.Action<object>> handlers = new Dictionary<string, System.Action<object>>();
    
    // 订阅
    public static void Subscribe(string messageType, System.Action<object> handler)
    {
        if (!handlers.ContainsKey(messageType))
            handlers[messageType] = handler;
        else
            handlers[messageType] += handler;
    }
    
    // 发布
    public static void Publish(string messageType, object data = null)
    {
        handlers[messageType]?.Invoke(data);
    }
    
    // 取消订阅
    public static void Unsubscribe(string messageType, System.Action<object> handler)
    {
        if (handlers.ContainsKey(messageType))
            handlers[messageType] -= handler;
    }
}
```

## 4.11 装饰器模式

#### 核心概念解析

**什么是装饰器模式？**

- 不改变原有类，通过包装扩展功能

### 装饰器实现

```csharp
// 1. 基础接口
public interface ICharacter
{
    void Attack();
    int GetPower();
}

// 2. 基础角色
public class Character : ICharacter
{
    public int power = 10;
    
    public void Attack() => Debug.Log("普通攻击");

}

// 3. 装饰器基类
public abstract class CharacterDecorator : ICharacter
{
    protected ICharacter character;
    
    public CharacterDecorator(ICharacter c) => character = c;
    
    public virtual void Attack() => character.Attack();
    public virtual int GetPower() => character.GetPower();
}

// 4. 具体装饰器
public class WeaponDecorator : CharacterDecorator
{
    private int addPower = 5;
    
    public WeaponDecorator(ICharacter c) : base(c) { }
    
    public override void Attack()
    {
        Debug.Log("装备武器，" + addPower + "点攻击力加成");
        base.Attack();
    }
    
    public override int GetPower() => base.GetPower() + addPower;
}
```
