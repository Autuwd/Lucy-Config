## 2) 单例模式与静态/全局状态

- 对应原文章节：UnityInterview/阶段四_设计模式.md -> 4.1 单例模式；以及阶段零/阶段五中的相关上下文。
- 练习目标：实现泛型 MonoBehaviour 单例、静态单例与非 MonoBehaviour 的两种实现方式，理解它们的适用场景与潜在风险。

### 练习清单
- 练习 A：泛型 MonoBehaviour 单例基类
  - 文件：Assets/Scripts/Singleton.cs
  - 框架代码：
```csharp
using UnityEngine;

public class Singleton<T> : MonoBehaviour where T : MonoBehaviour
{
    private static T _instance;
    private static readonly object _lock = new object();
    public static T Instance
    {
        get
        {
            if (_instance == null)
            {
                lock (_lock)
                {
                    if (_instance == null)
                    {
                        _instance = FindObjectOfType<T>();
                        if (_instance == null)
                        {
                            var obj = new GameObject(typeof(T).Name);
                            _instance = obj.AddComponent<T>();
                        }
                    }
                }
            }
            return _instance;
        }
    }
    protected virtual void Awake()
    {
        if (_instance == null)
        {
            _instance = this as T;
            DontDestroyOnLoad(gameObject);
        }
        else if (_instance != this)
        {
            Destroy(gameObject);
        }
    }
}
```

- 对应原文章节：UnityInterview/阶段一_CSharp基础.md -> 1.8 委托与事件；以及 UnityEvent 的使用（跨阶段应用）。
- 练习目标：实现简单的健康值变化通知系统，演示委托、事件和 UnityEvent 的用法与差异。

### 练习清单
- 练习 A：使用 C# 事件实现健康值变化通知
  - 文件：Assets/Scripts/HealthModel.cs
  - 框架：
```csharp
using System;

public class HealthModel
{
    public event EventHandler<int> OnHealthChanged;
    private int health = 100;
    public int Health
    {
        get => health;
        set
        {
            if (health != value)
            {
                health = value;
                OnHealthChanged?.Invoke(this, health);
            }
        }
    }
}

public class HealthUI : UnityEngine.MonoBehaviour
{
    public HealthModel model;
    void OnEnable()
    {
        model.OnHealthChanged += OnHealthChanged;
    }
    void OnDisable()
    {
        model.OnHealthChanged -= OnHealthChanged;
    }
    void OnHealthChanged(object s, int h)
    {
        UnityEngine.Debug.Log($"Health updated: {h}");
    }
}
```
- 练习 B：使用 UnityEvent 实现同样逻辑
  - 文件：Assets/Scripts/HealthUnityEvent.cs
  - 框架：
```csharp
using UnityEngine;
using UnityEngine.Events;

[System.Serializable]
public class HealthEvent : UnityEvent<int> { }

public class HealthEventPublisher : UnityEngine.MonoBehaviour
{
    public HealthEvent OnHealthChanged = new HealthEvent();
    private int health = 100;
    public void TakeDamage(int d)
    {
        health = Mathf.Max(0, health - d);
        OnHealthChanged.Invoke(health);
    }
}
```

## 4) 资源加载模型（Resources、AssetBundle、Addressables）

- 对应原文章节：UnityInterview/阶段三_Unity进阶.md -> 3.4 资源加载
- 练习目标：实现一个统一的资源加载接口，覆盖 Resources、AssetBundle、Addressables 三种加载方式。

### 练习清单
- 练习 A：Resources 加载封装
  - 文件：Assets/Scripts/ResourcesLoader.cs
  - 框架：
```csharp
using UnityEngine;
using System.Threading.Tasks;

public static class ResourcesLoader
{
    public static T Load<T>(string path) where T : UnityEngine.Object
    {
        return Resources.Load<T>(path);
    }
    public static async Task<T> LoadAsync<T>(string path) where T : UnityEngine.Object
    {
        ResourceRequest req = Resources.LoadAsync<T>(path);
        while (!req.isDone) await Task.Yield();
        return req.asset as T;
    }
}
```
- 练习 B：AssetBundle 简易封装（伪实现，示例结构）

  - 文件：Assets/Scripts/AddressablesLoader.cs
  - 框架：

```csharp
using UnityEngine;
using UnityEngine.AddressableAssets;
using UnityEngine.ResourceManagement.AsyncOperations;

public class AddressablesLoader : MonoBehaviour
{
    public void Load(string key)
    {
        Addressables.LoadAssetAsync<GameObject>(key).Completed += handle => {
            if (handle.Status == AsyncOperationStatus.Succeeded)
            {
                Instantiate(handle.Result);
            }
        };
    }
}
```
