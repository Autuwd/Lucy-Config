## 第一部分：编译与运行

### 方法 1：VS Code 编译（最推荐）

1. 打开 VS Code（终端运行 `code D:\Apps\SmallEngine\Projects\Raylib`）
2. 打开任意项目里的 `.cpp` 文件
3. 按 **`Ctrl+Shift+B`** → 回车 → 自动编译并运行

> VS Code 已配置好 `.vscode/tasks.json`，无需额外设置。

### 方法 2：`rl` 命令编译（命令行）

系统已安装全局 `rl` 命令，在任意文件夹的终端中：

```bash
rl main.cpp                  # 单文件
rl main.cpp player.cpp enemy.cpp  # 多文件
rl *.cpp                     # 所有 .cpp
```

> 编译器 `g++` 已加入系统 PATH，可以直接在 cmd/powershell 中使用。

### 方法 3：拖拽编译

把 `.cpp` 文件直接**拖到 `compile.bat` 图标上**松开 → 自动编译并运行

> 编译器会自动检测 `.c` 用 gcc，`.cpp` 用 g++

### 多文件项目编译

每个项目内都有一个 `compile.bat`（由 `new_project.bat` 自动生成），列出所有源文件：

```bat
@..\compile.bat main.cpp player.cpp enemy.cpp
```

然后在项目目录中执行 `compile.bat`，或按 Ctrl+Shift+B（VS Code）。

### 调试模式

`compile_debug.bat` 编译出来的程序会带一个控制台窗口，显示 `printf` 输出，方便调试：

```bash
compile_debug.bat 02_BouncingBall/main.cpp
```

## 第二部分：Raylib 核心概念

### 1. 游戏循环结构

所有 Raylib 程序都遵循这个结构：

```cpp
int main()
{
    // 1. 初始化
    InitWindow(800, 600, "标题");
    SetTargetFPS(60);

    // 2. 游戏循环
    while (!WindowShouldClose())
    {
        // 一帧内的逻辑
        float dt = GetFrameTime();  // 帧时间

        // --- 更新阶段 ---
        UpdateGame(dt);

        // --- 绘制阶段 ---
        BeginDrawing();
        ClearBackground(RAYWHITE);
        DrawEverything();
        EndDrawing();
    }

    // 3. 清理
    CloseWindow();
    return 0;
}
```

### 2. 坐标系

```
(0,0) ──────────────→ X 正方向
  │
  │
  ↓
Y 正方向
```

- 原点在**左上角**
- X 向右，Y 向下
- 单位：像素

### 3. 帧时间 (Delta Time)

```cpp
float dt = GetFrameTime();  // 单位：秒
```

**为什么需要 dt：** 不同电脑帧率不同。用 `dt` 乘以速度可以保证在不同帧率下移动速度一致。

```cpp
// ❌ 错误：60FPS 下每秒走 60*5=300px，但 30FPS 下只有 150px
player.x += 5;

// ✅ 正确：无论多少帧，每秒都走 speed × 1 秒
player.x += speed * dt;  // speed = 300 → 每秒移动 300px
```

### 4. Vector2 和 Rectangle

```cpp
// 位置/方向
Vector2 pos = { 100.0f, 200.0f };
Vector2 dir = { 1.0f, 0.0f };

// 矩形区域
Rectangle rect = { x, y, width, height };
```

## 第三部分：编译错误排查

### 常见错误

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `'g++' is not recognized` | 编译器不在 PATH 中 | 用 compile.bat 编译，或手动将 `C:\raylib\w64devkit\bin` 加入 PATH |
| `fatal error: raylib.h: No such file or directory` | 找不到 raylib 头文件 | 确保编译命令有 `-I"C:\raylib\raylib\src"` |
| `undefined reference to 'InitWindow'` | 未链接 raylib 库 | 确保有 `-lraylib -lopengl32 -lgdi32 -lwinmm` |
| `collect2.exe: error: ld returned 1 exit status` | 链接错误 | 通常是缺少某个库，或 main 函数写错 |
| 窗口一闪而过 | 程序运行完立刻退出了 | 检查是否有 `while (!WindowShouldClose())` 循环 |

### 代码常见问题

```cpp
// ❌ 忘记 pair
BeginDrawing();
DrawText("Hi", 10, 10, 20, BLACK);
// 漏了 EndDrawing();

// ✅ 正确
BeginDrawing();
DrawText("Hi", 10, 10, 20, BLACK);
EndDrawing();

// ❌ C++ 中结构体初始化需要类型匹配
Vector2 v = { 100, 200 };  // 隐式转换 int → float，编译有警告

// ✅ 最好写 float
Vector2 v = { 100.0f, 200.0f };
```

## 第四部分：C++ 在 Raylib 中的注意事项

### 1. 为什么用 `.cpp` 而不只是 `.h`？

```cpp
// Ball.h  — 声明（告诉编译器"有这个东西"）
class Ball {
    void Update(float dt);   // 声明：没有函数体
};

// Ball.cpp — 定义（实现）
#include "Ball.h"
void Ball::Update(float dt) { // 定义：有函数体
    position.x += velocity.x * dt;
}
```

**VS Unity：** C# 不需要分离 .h/.cpp 因为 .NET 的元数据系统把类型信息嵌入了 .dll。C++ 的 `.h` 文件本质是"手动维护的类型元数据"——这就是为什么编译 C++ 比 C# 慢：编译器需要反复解析同样的头文件。

### 2. 编译模型（理解为什么 C++ 比 C# 慢）

```
C++:       main.cpp → #include "Ball.h" → 预处理展开 → 编译 → main.obj
           Ball.cpp → #include "Ball.h" → 预处理展开 → 编译 → Ball.obj
                                       ↓
                                  链接 → main.exe

C#:        Program.cs → 编译器直接引用 Ball.cs 的元数据 → 一次编译
```

- 每个 `.cpp` 是独立的**编译单元**
- 修改一个 `.h` 文件 → 所有 `#include` 它的 `.cpp` 都要重新编译
- Unity 的 C# 编译器不需要这种分离——所以你修改一个类，Unity 只需要重新编译那一个文件

### 3. C++ 特性使用建议

| C++ 特性 | 推荐程度 | Unity 对应 | 说明 |
|----------|----------|-----------|------|
| `class` / `struct` | ✅ 推荐 | `MonoBehaviour` | 游戏对象封装 |
| 构造函数/析构函数 | ✅ 推荐 | `Awake()` / `OnDestroy()` | 生命周期管理 |
| `std::vector` | ✅ 推荐 | `List<T>` | 动态数组，扩容机制相同 |
| `const` 方法 | ✅ 推荐 | 无直接对应 | `void Draw() const` 承诺不修改对象 |
| 引用 `&` | ✅ 推荐 | `ref` 参数 | 避免拷贝，C# 默认引用类型 |
| `enum class` | ✅ 推荐 | `enum` | 类型安全—UdonSharp 的枚举坑就是这个 |
| `new` / `delete` | ✅ 推荐 | `new` + GC | C++ 手动管理，C# GC 自动回收 |
| 模板 | ⚠️ 适度 | 泛型 (Generics) | C# 泛型在运行时实例化，C++ 模板在编译时 |
| 继承/虚函数 | ⚠️ 适度 | `virtual` / `override` | C++ 虚函数表 (vtable) 和 C# 一致 |
| 异常 | ❌ 避免 | `try-catch` | 游戏性能敏感，关闭异常是惯例 |

### 4. 关键区别：值类型 vs 引用类型

```cpp
// C++: struct/class 都可以在栈或堆
Player p1;            // 栈——自动销毁（对应 C# 的 struct）
Player* p2 = new Player();  // 堆——手动 delete（对应 C# 的 class）

// C#: struct 是值（栈），class 是引用（堆）
Player p1 = new Player();  // 堆——GC 自动回收
```

C++ 中你必须自己决定对象放在栈还是堆。Unity 中你只需要决定用 struct 还是 class——底层逻辑和 C++ 一样。

### 5. float 字面量

```cpp
// C++ 中整数会隐式转换为 float，但会有 warning
Vector2 pos1 = { 100, 200 };           // warning:  narrowing conversion

// 最好显式写 float
Vector2 pos2 = { 100.0f, 200.0f };     // ✅ 推荐
```

## 第七部分：常见问题 (FAQ)

### Q: 编译后窗口一闪而过看不到效果？

A: 确保代码中有 `while (!WindowShouldClose())` 循环。或者用 `compile_debug.bat` 编译，会保留控制台窗口。

### Q: 如何显示中文？

A: Raylib 默认字体不支持中文，需要加载中文字体：

```cpp
Font cnFont = LoadFontEx("C:/Windows/Fonts/msyh.ttc", 32, 0, 0);
DrawTextEx(cnFont, u8"你好世界", { 10, 10 }, 28, 2, WHITE);
```

### Q: 如何使用外部图片/音频资源？

A: 在项目文件夹下创建 `resources/` 目录，把文件放进去：

```
MyProject/
├── main.cpp
├── resources/
│   ├── player.png
│   ├── bgm.mp3
│   └── font.ttf
```

然后代码中相对路径引用：

```cpp
Texture2D tex = LoadTexture("resources/player.png");
Music bgm = LoadMusicStream("resources/bgm.mp3");
```

### Q: 如何在 VS Code 中编译？

A: 打开项目文件夹，按 **Ctrl+Shift+B** → 回车。或者终端运行 `rl main.cpp`。

### Q: `rl` 命令是什么？

A: 一个全局快捷编译命令，在 `C:\raylib\bin\rl.cmd`。原理等同 `compile.bat`：
```bash
rl main.cpp
# 等价于：
g++ main.cpp -o main.exe -I"C:\raylib\raylib\src" -L"C:\raylib\raylib\src" -lraylib -lopengl32 -lgdi32 -lwinmm -std=c++17
```

### Q: 多个 .cpp 文件怎么组织？

A: 在项目内创建 `compile.bat`，列出所有源文件：

```bat
@..\compile.bat main.cpp player.cpp enemy.cpp
```
